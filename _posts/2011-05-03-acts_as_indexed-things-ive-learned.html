Acts_as_indexed is a great utility for powerful, zero-dependency, full-text search on the Ruby on Rails framework.  Currently in its 0.7.2 release, this gem can be attached to any model to add search indexing to the data.  You can read more on the setup at https://github.com/dougal/acts_as_indexed.

Here are a couple of things I couldn't find in the documentation.

Resetting the index.  

Say for any reason the index becomes corrupted or starts showing wrong data because the data in your database make a sudden change unexpectedly. You have to delete the /index directory that is automatically created. Just as simple as typing in your rails_root folder

rm -rf index/

This remove your index folder and when you run your app again this will reindex the whole thing and the indexing will be as good as new. This is pretty important when working in development as the data can change often.  You may want to consider turning this into a rake task as well.

Version Control (Git, SVN, etc.)

Make sure to ignore index/. For git in your rails_root folder you would type:

echo index/ >> .gitignore

This will append index/ to the end of .gitignore.

There really isn't any reason to keep the index of your development data and your production or testing data in your repo. This isn't code and can vary widely from one setup to the next.

I want to thank Douglas F. Shearer for putting together such a useful Gem. Again you can read more about this at https://github.com/dougal/acts_as_indexed.
