Someone may come along and tell me otherwise, but as of right now I have no idea how to pass a helper into a layout/template and yield the content in that layout.  I've tried a number of different ways of using the  <%= render %>, but it seems to be a fruitless endeavor.

This may sound wildly confusing, so let me explain.  I've recently come across a problem at work where many of the things on our website are within a specific container, ie a box with a header over top and some special formatting for it.  So like any developer that likes to keep things DRY as possible, I thought, hell, why not just have some sort of default, make it a layout and just pass whatever html, whether it be partial or helper, to the layout. This way I don't have to keep on retyping everything.

The idea was sound, and in practice it works when dealing with partials.  You can easily call the command in haml using

= render :partial => "/link/to/partial", :layout => "/my/sweet/layout"

And this works, but when you have html that being generated by a helper, it isn't possible to use any number of render setups (at least none that I've found) to pass that helper into the layout and output what you need.

My short term solution, which may end up being my long term, is to setup a helper method within my main application that accepts html in the base and outputs the properly formatted html.  I hate this solution, because if I want to pass partials into this, I'm going to have to use the helper capture_html to my html to pass into the helper.  This seems like a lot of steps.

Anyways, I'm off to research this a bit more. Hopefully I can find something a bit more meaningful.
