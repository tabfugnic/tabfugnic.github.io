So I'm not going to lie, I spent the better part of the morning trying to figure out how the hell I could resize my LVM2 Partition. I scoured the interwebs looking for answers, reading blog posts and what have you. To which I finally found enough information to get what I needed. I've decided to share this information with you.

Let me establish that the scenario I will present to you will be just an example. Some variables and paths will need to be changed according to your own machine. This also assumes you have some extra partition that isn't currently being allocated.

Also, you can read more about this from the following webpages:

  • http://www.howtoforge.com/logical-volume-manager-how-can-i-extend-a-volume-group
  • http://www.tcpdump.com/kb/os/linux/lvm-resizing-guide/expand.html
  • http://tldp.org/HOWTO/LVM-HOWTO/extendlv.html

DISCLAIMER: Perform this task at your own risk. Changing around partitions is dangerous and can prevent you from accessing your data and make it completely unrecoverable. If you do this, please don't for get to create BACKUPs.

Our first task is to use that unallocated space and create another virtual partition.
As either root or preferably as sudo, type:

fdisk /dev/sda

/dev/sda is interchangeable with whatever hard disk you have.
You should see a screen come up with the prompt that looks like this.

Command (m for help):

This will be the command prompt we'll use to create a new partition. Type:

Command (m for help): n
l logical
p physical
p

Assuming you want to use all of the unallocated space, just use the defaults.
Now to change it over to a LVM, type:


Command (m for help): t
Partition number (1-3): 3 *This is the drive number that was created for me*
Hex code (type L to list codes): 8e
Command (m for help): w

The partition has been allocated and now needs to have a few finishing touches for the next steps.

pvcreate /dev/sda3 *Again, change depending on what you have.*

Get the name of your Virtual Group that you'll be extending by typing

vgdisplay

With that name type the command:

vgextend /dev/nameOfVG /dev/sda3

You've now extended the group, but now comes the part where you need to extend the actual LVM.

Let's assume we want to extend our drive by 10 Gigs. Type the commands

lvextend -L +10G /dev/NameOfVG/root

With the drive extended we now have to fill it in with ext2/ext3 file formatting. Type the commands

resize2fs -p /dev/NameOfVG/root

It will inevitably complain about it being online, but will extend it regardless. If you are using a different type of file system other than ext2 then I suggest you refer to the third link. It gives detailed instructions on how to use the other file types.

There you have now just extended your drive by 10gigs.
